## Summary

Fixed three critical bugs in token budget enforcement for `mdtldr context --tokens N`:

### 1. Orphaned children bug (src/summarize/summarizer.ts:306-343)
**Problem:** If a parent section was truncated, all children were lost even if they would fit.

**Fix:** Implemented "orphan rescue" - when a parent doesn't fit the budget, we now still traverse its children and rescue any that fit. These rescued children are promoted to the parent's level in the output tree.

### 2. Token estimation inaccuracy (src/utils/tokens.ts:42-75)
**Problem:** The `4 chars/token` approximation could be ±30% off, especially for code.

**Fix:** Improved the approximation with content-aware heuristics:
- Detects code blocks and uses 3.5 chars/token for code (more symbols/short identifiers)
- Accounts for punctuation that often becomes separate tokens
- Accounts for whitespace collapsing
- Adds a 10% safety margin to avoid under-counting

### 3. Formatting overhead under-estimated (src/summarize/summarizer.ts:286-298)
**Problem:** Hardcoded 50-token reserve was insufficient for long paths/titles.

**Fix:** Dynamically calculates formatting overhead based on actual content:
- Title length
- Path length  
- Token count line
- Topics line (if present)
- Truncation warning (reserved in case needed)
- 10-token safety margin

### Test updates
Updated `src/search/searcher.test.ts` to use a dynamic token limit that ensures the test works correctly with the new more accurate token counting.
